Controlled Blink

Key Ideas

  • Input from GPIO port.
  • Output to GPIO ports.

Description

Control a pattern of blinking LEDs based on digital input.

Compatability

ADAPT Core Hardware v 1.04, eMote release v 1.01

Complexity Level

Introductory.

Setup

TThe ADAPT platform has several GPIO (General Purpose I/O) ports. Each can be configured as input or as output. The available ports are as follows. The first column gives the CPU pin, the second gives the labeling on the ADAPT Development board and the third is the usage in this app note.

CPU Pin ADAPT Dev Board Labeling This App Note
58 GPIO01 Led1
55 GPIO02 Led2
53 GPIO03 Led3
52 GPIO04 SignalHigh
51 GPIO05 Input

Make sure LEDs are connected to the CPU pins indicated. Instructions on how to connect an LED can be found online; for example: www.instructables.com/id/LEDs-for-Beginners . Be sure you use a suitable resistor to avoid overloading the port. Connect a switch between SignalHigh and Input. This can be a push button, an on-off switch, or another kind of switch that lets you connect and disconnect Signal High to/from Input. Or you can use the digital output of some other device.

For information on deployment and on monitoring console output (including the output of Print.Debug statements), see the eMote user guide.

Discussion

The app note defines the three LED ports in the array Leds. It defines an output SignalHigh which is initialized to high (true). This provides a high signal for logic purposes. The Input port is defined using Port.ResistorMode.PullDown so that when nothing is connected, it will be low.

The app note initializes toggleLight to true. The main loop reads the input. If high (true), it sets the LED port values to current value of toggleLight in order from Led1 to Led3; if low (false), it sets the LED port values in the reverse order. After each loop, it changes the value of toggleLight. So the first time through it turns the lights on and the second time through it turns them off.

When the program is running, use your switch to ensure that SignalHigh and Input are disconnected. The value read from input should be low (false) and the second loop should be executed. Then use the switch to connect SignalHigh and Input. The value read should be high (true ) and the first loop should be executed.
